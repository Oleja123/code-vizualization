# Selenium E2E — описание тестов

## Общая информация

- **Фреймворк:** Selenium WebDriver 4.21 + JUnit 5
- **Паттерн:** Page Object Model
- **Браузер:** Google Chrome (один на весь прогон)
- **Запуск:** `mvn test` (или `mvn test -Dheadless=true` для CI)

---

## Последовательность шагов

### Шаг 1 — Открытие приложения показывает форму авторизации

**Что делает:**
Открывает `http://localhost:3000` без авторизации.

**Что проверяет:**
Форма «Вход в систему» отображается по умолчанию.

**Состояние сессии:** нет → нет

---

### Шаг 2 — Регистрация нового пользователя

**Что делает:**
1. Переключает форму в режим «Регистрация»
2. Вводит уникальный логин `e2e_<timestamp>` и пароль
3. Отправляет форму

**Что проверяет:**
- Заголовок формы переключился на «Регистрация»
- После регистрации Vue автоматически логинит пользователя
- Имя пользователя отображается в шапке приложения

**Состояние сессии:** нет → **открыта** (переходит в шаг 3)

---

### Шаг 3 — Выход и повторный вход

**Что делает:**
1. Использует сессию от шага 2 (хедер уже виден)
2. Нажимает кнопку «Выйти»
3. Входит снова с теми же учётными данными

**Что проверяет:**
- После выхода отображается форма «Вход в систему»
- После повторного входа имя пользователя снова в шапке

**Состояние сессии:** открыта → выход → **открыта снова** (переходит в шаг 4)

---

### Шаг 4 — Трассировка схемы

**Что делает:**
1. Использует сессию от шага 3 (логин не нужен)
2. Переходит на вкладку «Трассировка схемы»
3. Нажимает «Сгенерировать схему»
4. Нажимает «Начать трассировку»
5. Делает два шага вперёд («Вперёд →»)
6. Нажимает «Стоп»

**Что проверяет:**
- SVG блок-схема отображается после генерации
- Кнопка «Начать трассировку» доступна после генерации
- После остановки кнопка «Начать трассировку» снова появляется

**Состояние сессии:** открыта → **открыта** (переходит в шаг 5)

---

### Шаг 5 — Метрики

**Что делает:**
1. Использует сессию от шага 4 (логин не нужен)
2. Переходит на вкладку «Метрики»
3. Нажимает «Подсчитать метрики» (код уже загружен)
4. Ожидает результат до 60 секунд

**Что проверяет:**
- Сообщение «✓ Метрики подсчитаны» отображается
- На экране есть хотя бы одна карточка функции
- Бейдж с количеством функций не пустой

**Состояние сессии:** открыта → **открыта** (переходит в шаг 6)

---

### Шаг 6 — Выход: форма авторизации снова отображается

**Что делает:**
1. Использует сессию от шага 5
2. Нажимает «Выйти»

**Что проверяет:**
- Форма «Вход в систему» снова отображается
- Хедер приложения (`.app-header`) полностью исчезает с экрана

**Состояние сессии:** открыта → **закрыта**

---

### Шаг 7 — Перезагрузка страницы не восстанавливает сессию

**Что делает:**
1. Логинится заново
2. Выходит из системы
3. Делает `refresh` страницы

**Что проверяет:**
Сессия не восстанавливается после перезагрузки — форма входа снова видна.
Проверяет что бэкенд не использует восстанавливаемые сессии (например, «запомнить меня»).

**Состояние сессии:** нет → логин → выход → refresh → **нет**
