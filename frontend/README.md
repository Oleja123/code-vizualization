# Code Visualization Frontend

Фронтенд приложение для визуализации пошагового выполнения C-кода.

## Структура проекта

```
frontend/
├── src/
│   ├── api/
│   │   └── interpreter.js      # API клиент для взаимодействия с interpreter-service
│   ├── components/
│   │   ├── Array.vue            # Компонент визуализации массива
│   │   ├── Array2D.vue          # Компонент визуализации двумерного массива
│   │   ├── CodeEditor.vue       # Редактор кода с кнопками управления
│   │   ├── RuntimeVisualization.vue # Главный компонент визуализации
│   │   ├── Scope.vue            # Компонент области видимости
│   │   ├── StackFrame.vue       # Компонент stack frame
│   │   └── Variable.vue         # Компонент переменной
│   ├── views/
│   │   └── VisualizationView.vue # Главная страница визуализации
│   ├── App.vue                  # Корневой компонент
│   ├── main.js                  # Точка входа
│   └── style.css                # Глобальные стили
├── index.html
├── package.json
└── vite.config.js
```

### 4) Анализ кода (cppcheck)

- Отдельная вкладка «Анализ кода».
- Отправка C-кода в `cppcheck-analyzer-service` (`POST /api/analyze`).
- Список найденных issues c сообщением и строкой.
- Подсветка строк с проблемами и фокус по клику на issue.

## Установка и запуск

1. Установите зависимости:
```bash
cd frontend
npm install
```

2. Запустите dev-сервер:
```bash
npm run dev
```

Приложение будет доступно по адресу http://localhost:3000

## Функциональность

### Редактор кода
- Поле для ввода C-кода с подсветкой номеров строк
- Подсветка текущей выполняемой строки (желтый фон)
- Кнопка "Выполнить" для компиляции и запуска кода
- После успешного выполнения появляются:
  - Кнопки "Шаг вперед" и "Шаг назад" для пошагового выполнения
  - Кнопка "Заново" для повторного выполнения
- Отображение текущего шага и общего количества шагов
- Отображение ошибок компиляции/выполнения

### Визуализация состояния
- **StackFrame**: Отображение стека вызовов функций с именами функций и их областями видимости
- **Scope**: Области видимости с переменными, массивами и двумерными массивами
  - Для global фрейма показываются все scope'ы
  - Для остальных фреймов первый scope (параметры) отображается отдельно
- **Variable**: Переменные с именем и значением (или "?" если значение не определено)
- **Array**: Одномерные массивы с индексами и значениями элементов
- **Array2D**: Двумерные массивы с визуализацией в виде матрицы

### Подсветка изменений
Переменные и элементы массивов подсвечиваются желтым цветом, если они были изменены на предыдущем шаге (когда `step_changed === current_step - 1`).

## API

Приложение взаимодействует с `interpreter-service` через прокси:
- Endpoint: `POST /api/snapshot`
- Запрос: `{ code: string, step: number }`
- Прокси настроен в vite.config.js для перенаправления запросов на http://localhost:8080
